CityServer

This item only works when running on the server. Server

This class holds the data of every city on the island.

DANGER

Changing most of the object's properties directly may lead to unexpected behavior. Most of the properties should be only read and changed only through the object's methods.

Properties​

ID​

This item is read only and cannot be modified. Read Only
</>
CityServer.ID: number

The identification property of the city.

Island​

This item is read only and cannot be modified. Read Only
</>
CityServer.Island: IslandServer

The island the city is associated with.

TIP

Use this property if you have a city and you need to interact with the island the city is on.

Type​

This item is read only and cannot be modified. Read Only
</>
CityServer.Type: ConfigTypes.CityType

What type the city is.

Config​

This item is read only and cannot be modified. Read Only
</>
CityServer.Config: ConfigTypes.CityConfig

The city's config.

Position​

This item is read only and cannot be modified. Read Only
</>
CityServer.Position: Vector3

The position the city is located at. This should be equal to city.Node.Position

Node​

This item is read only and cannot be modified. Read Only
</>
CityServer.Node: NodeServer

The node the city's associated with. Every city has exactly one node, even if it isn't connected to any road yet. The city acts only as an addition to the node, making it special compared to its basic form.

ConnectedToHQ​

</>
CityServer.ConnectedToHQ: boolean

True if the city's node has a connection to the island's HQ.

This is set externally (from the island when a new road connection appears).

Spawning​

This item is read only and cannot be modified. Read Only
</>
CityServer.Spawning: boolean

True if the city is currently running the loop to spawn packages, false otherwise.
