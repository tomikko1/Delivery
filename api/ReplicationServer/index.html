ReplicationServer

This item only works when running on the server. Server

This class handles the basic common needs of replication of any object.

WARNING

This class doesn't handle all the replication that needs to be done. It handles only the basic common needs, meaning storing the players that the object should be replicated to, replicating the object on creation and replicating destroy method with functions that are provided to it.

It uses this logic to handle adding and removing players to and from replication. Simply, this means that every time some player is removed from the replication, this class will invoke the :Destroy() method on the client as if the object was destroyed, even though it remains living on the server.

DANGER

Changing any of the object's properties directly may lead to unexpected behavior. All of the properties should be only read and changed only through the object's methods.
